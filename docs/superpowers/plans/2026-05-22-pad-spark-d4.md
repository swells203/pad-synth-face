# PAD Spark D4 Extension Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a fourth data tier D4 (Set A = 16,384 / Set B = 32,768 samples) to the just-merged PAD scaling sweep, run 9 new cells on the DGX Spark, and append the result to the existing report — confirming or qualifying the data-limited diagnosis.

**Architecture:** Two new generation configs (same seeds as D3 so D4 is a same-distribution superset). `scripts/spark_sweep.py` extends `DATA_LEVELS` to include `D4` and accepts `--set-a-d4`/`--set-b-d4` args. The 9 D4 result JSONs rsync into the existing report directory; the combined `summary.csv` regenerates from all 36 JSONs; the existing report gains a "D4 update" section and the decisions/roadmap doc gains a one-line append.

**Tech Stack:** Same as the parent project — Python 3.11+ (laptop) / 3.12 (Spark), PyTorch nightly cu128, torchvision, numpy, Pillow, pyyaml, pytest with `--import-mode=importlib`. No new packages.
